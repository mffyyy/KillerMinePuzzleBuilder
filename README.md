# Killer MineDoku Puzzle Builder

本目录是一个无依赖的本地网页出题工具。

## 打开方式

直接打开 `index.html`，或在本目录启动任意静态服务器。

```powershell
python -m http.server 8777 --directory PuzzleBuilder
```

然后访问 `http://127.0.0.1:8777/`。

## 基本流程

1. 选择棋盘尺寸和每行/列雷数。
2. 点击棋盘格子进行选择；按住 `Shift` 可拖拽连选。
3. 输入 cage 雷数，点击“生成 Cage”。
4. 点击已有 cage 可重新选中，点击“更新雷数”可修改提示。
5. 选择空白格后点击“加入当前 Cage”，可把格子并入列表里选中的 cage；如果选择中包含已归属 cage 的格子，会优先把该 cage 作为目标。
6. 选择已分配格后点击“踢出 Cage”，可把格子从原 cage 中移除。
7. 点击“标雷/取消”可给选择格添加红色 X 标记，用来检查答案和提示冲突；点击“清空标雷”可一键清除。
8. 点击“验证唯一解”检查是否唯一可行；如果出现多解，棋盘上方会出现切换按钮，可逐个查看找到的解。
9. 点击“导出 JSON”保存题面数据。

棋盘上方会实时显示：

- 总共需要多少雷。
- Cage 提示已经布设多少雷，以及还差多少。
- 红 X 标雷已经布设多少雷，以及还差多少。
- 标雷与行、列、cage 提示之间的冲突。

## 快捷键

- `0-9`：用当前选择生成 cage，并把按下的数字作为 cage 雷数。
- `I`：把选择中的空白格加入当前 cage。
- `O`：把选择格从所在 cage 踢出。
- `M`：标雷/取消标雷。
- `Esc`：清空选择。
- `Shift` + 拖拽：连续选择/取消多个格子。

## 多解展示

验证器会最多收集 12 个解用于展示。如果不是唯一解，可以用“上一解 / 下一解”在棋盘上切换，当前展示的解会用深蓝色 X 标出。手动标雷仍然使用红色 X。

## JSON 结构

坐标使用 1-based 行列，便于和题面图、Unity 数据对齐。

```json
{
  "name": "killer_minedoku_custom",
  "size": 9,
  "minesPerRow": 1,
  "minesPerColumn": 1,
  "markedMines": [[1, 3]],
  "cages": [
    {
      "id": 1,
      "clue": 0,
      "cells": [[1, 1], [1, 2]]
    }
  ]
}
```

验证器会检查：

- 所有格子是否都被 cage 覆盖。
- cage 雷数是否合法。
- 是否满足每行/每列固定雷数。
- 满足全部 cage 约束的解是否唯一。
