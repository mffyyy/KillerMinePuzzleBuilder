# Killer MineDoku Puzzle Builder

一个无依赖的本地网页出题工具，用于制作 Killer MineDoku / Minesweeper 风格的 cage 题面。当前版本采用 Neobrutalist 视觉风格，支持独立行列雷数、cage 编辑、标雷检查、唯一解验证和 JSON 导入导出。

## 打开方式

可以直接打开 `index.html`，也可以在本目录启动一个静态服务：

```powershell
python -m http.server 8778
```

然后访问：

```text
http://127.0.0.1:8778/
```

## 基本流程

1. 选择宫格数，设置默认行/列雷数，点击“新建”生成棋盘。
2. 也可以在“行雷数 / 列雷数”里单独调整每一行、每一列的雷数。
3. 点击格子进行选择；按住 `Shift` 可连续选择或取消多个格子。
4. 输入当前雷数，点击“生成 Cage”创建 cage。
5. 点击 cage 列表里的项目可激活 cage；激活后边界由虚线变为实线。
6. 选择格子后，可用“更新雷数”“加入 Cage”“踢出 Cage”编辑当前 cage。
7. 在“地雷标记”里点击“标记/取消”，或按 `M`，可切换选中格子的地雷标记。
8. 点击“验证唯一解”检查题面是否唯一；多解时可在棋盘左上方切换上一解、下一解或隐藏解。
9. 使用“导入 JSON / 导出 JSON”读取或保存题面数据。

## 视觉与交互

- Cage 的颜色会同步到棋盘虚线、cage 数字和右侧 cage 列表标识。
- `0` cage 使用绿色；其他 cage 不使用黄色，避免和格子选择高亮冲突。
- 选中单格时显示黄色叠层；选中 cage 时只将 cage 边界从虚线切换为实线。
- 地雷、剩余可放、已验证等状态使用统一的图标卡片样式。

## 快捷键

- `0-9`：生成对应雷数的 Cage。
- `I`：加入 Cage。
- `O`：踢出 Cage。
- `M`：标记地雷/取消。
- `Esc`：清空当前选择。
- 按住 `Shift`：连选。

## JSON 结构

坐标使用 1-based 行列，方便和题面图、Unity 数据对齐。

```json
{
  "name": "killer_minedoku_custom",
  "size": 9,
  "minesPerRow": 1,
  "minesPerColumn": 1,
  "rowMines": [1, 1, 1, 1, 1, 1, 1, 1, 1],
  "columnMines": [1, 1, 1, 1, 1, 1, 1, 1, 1],
  "markedMines": [[1, 3]],
  "cages": [
    {
      "id": 1,
      "clue": 0,
      "cells": [[1, 1], [1, 2]]
    }
  ]
}
```

## 验证规则

验证器会检查：

- 所有格子是否都被 cage 覆盖。
- 每个 cage 的雷数是否合法。
- 行雷数之和与列雷数之和是否一致。
- 标记地雷是否超出行、列或 cage 限制。
- 满足全部行、列、cage 约束的解是否唯一。

验证器最多保留 12 个解用于展示。当前展示的解会以紫色 `X` 标出，手动标雷仍使用红色地雷图标。

## 文件说明

- `index.html`：页面结构。
- `styles.css`：Neobrutalist 视觉样式、棋盘、侧栏和交互状态。
- `app.js`：棋盘状态、cage 编辑、标雷、JSON 导入导出和唯一解验证逻辑。
- `assets/`：状态图标、地雷图标、垃圾桶图标等 UI 资源。
- `previews/`：视觉迭代预览截图。
